
Интернет-магазин "Курский двор"

Верстка полноценного интернет-магазина, со страницами:
- Главная
- О компании
- Каталог
- Детальная карточка товара
- Избранное
- Личный кабинет
- Оформление заказа
- Поиск
- Корзина
- Калькулятор
- Стр 404

Сайт реализован на чистом Java Script
Библиотеки применяемые на сайте так же написаны на Java Script

При верстке сайта использованы относительные единицы REM
В медиазапросах применяются относительные единицы EM

Отступы реализованы через not-child
Текстовые отступы от заголовков, через not-child в еденицах em относительно шрифта заголовка. Уменьшается шрифт, автоматически уменьшается отступ.
Межстрочный интервал line-height в процентном соотношении относительно шрифта.

Сетка сайта адптивная, резиновая. Реализована на флексах и грид-технолгиях, с минимальным колличеством медиазапросов.

Сайт написан на технолгии SCSS, по методологии БЭМ, компонетно. Любые блоки на сайте можно менять местами и переиспользовать.

Шапка сайта построена гибко с возможностью менять элементы местами, переключать и убирать во все. 
Построена архитектра управления шапкой из админки CMS.

Реализована самописная анимация добавления товара в корзину.
При клике на кнопку В корзину, в карточке товара или в детальной странице товара, картинка товара, а так же название и цена,
плавно улетают, уменьшаясь при этом в шапку в элемент Корзина. При клике на Корзину в шапке, откроется меню с превью добавленного товара, его названием и ценной, 
а так же колличеством. Если все товары удалить из меню, кликнув на крестик, то меню отображатся не будет до следующего добавления товара.

Так же реализован функционал Показать еще на странице Каталог категория и Распродажа. Создан json-файл с товарами и при помощи ajax технологии,
при клике на кнопку Показать еше, подгружаются дополнительные товары, без перезагрузки страницы.

Реализован функционал перелиствания изображений в карточке товара при наведении на область картинки.

Иконки реализованы через svg-спрайты и иконочные шрифты.

Установка и работа с шаблоном (с правами администратора!):
В терминале:
1) npm i
2) gulp 
Шаблон начнет работать в режиме разработчика, без сжатия картинок и кода.

Когда верстка закончена и пришло время грузить на сервер/отдавать заказчику:
В новом окне терминала:
1) gulp build
Выполнится оптимизация кода и картинок. 
После завершения действий галпом, папка с результатом готова к выгрузке на сервер/передаче заказчику

Внимание! 
После выполнения gulp build создаются webp файлы картинок. Также они интегрируются в html и css. 
Следовательно, стоит проверить все ли корректно отображается и работает.
Если после выполнения gulp build файлы редактировались, действие (gulp build) нужно повторить.

Возможные проблемы и их решения:
--------------------------------------
Ошибка node-sass.
Решения:
npm rebuild node-sass
и/или
npm install sass gulp-sass --save-dev
--------------------------------------
Ошибка Pyton
Решени:
npm install --global windows-build-tools
--------------------------------------
Ошибка imagemin.
Варианты решения :
- папки и файлы должны быть названы латиницей без пробелов
- тег img и его содержимое должны быть записаны в одну строку без переносов
- в атрибуте src должен быть указан путь к существующей картинке
